## 一、环境准备

### 📍 执行位置：三台都要做

**设置主机名**（在哪台就设哪个名）：

```bash
# 📍 .10 docker-master
hostnamectl set-hostname docker-master

# 📍 .20 docker-node1
hostnamectl set-hostname docker-node1

# 📍 .40 docker-node2
hostnamectl set-hostname docker-node2
```

**统一 hosts 文件**（📍 .10 .20 .40 都执行）：

```bash
cat >> /etc/hosts << 'EOF'
192.168.133.10 docker-master
192.168.133.20 docker-node1
192.168.133.40 docker-node2
EOF
```

### 📍 执行位置：.10 docker-master

**生成密钥 + 分发**：

```bash
# 生成
ssh-keygen -t rsa -b 2048 -f /root/.ssh/id_rsa -N ""

# 分发（需要输入对方 root 密码）
ssh-copy-id root@docker-node1   # 输入 root
ssh-copy-id root@docker-node2   # 输入 root

# 验证免密
ssh root@docker-node1 "hostname"  # 应输出 docker-node1
ssh root@docker-node2 "hostname"  # 应输出 docker-node2
```

## 二、Docker 安全配置

### 📍 执行位置：.20 docker-node1 和 .40 docker-node2（两台都要做）

**1. Docker 守护进程加固**：

```bash
cat > /etc/docker/daemon.json << 'JSON'
{
  "icc": false,
  "live-restore": true,
  "userland-proxy": false,
  "no-new-privileges": true,
  "log-driver": "json-file",
  "log-opts": { "max-size": "10m", "max-file": "3" }
}
JSON
systemctl daemon-reload
systemctl restart docker
```

| 参数 | 作用 |
|------|------|
| icc=false | 禁止容器间直接通信 |
| no-new-privileges | 防止 setuid 提权 |
| live-restore | Docker 重启时容器保活 |

**2. 启动安全容器**：

```bash
docker rm -f webapp 2>/dev/null
docker run -d --name webapp \
  --memory="256m" \
  --cpus="0.5" \
  --cap-drop=ALL \
  --cap-add=NET_BIND_SERVICE \
  --read-only \
  --tmpfs /tmp:rw,noexec,nosuid,size=64M \
  --tmpfs /run:rw,noexec,nosuid,size=64M \
  --tmpfs /var/cache/nginx:rw,noexec,nosuid,size=64M \
  --security-opt=no-new-privileges \
  -p 8080:80 \
  nginx:alpine
```

| 参数 | 安全效果 |
|------|----------|
| --memory=256m | 防 OOM 耗尽宿主机 |
| --cap-drop=ALL | 移除所有内核能力 |
| --read-only | 文件系统不可改 |
| --tmpfs noexec,nosuid | 临时目录禁止执行 |

**3. 验证**：

```bash
docker ps
docker inspect webapp --format 'CapDrop:{{.HostConfig.CapDrop}} ReadOnly:{{.HostConfig.ReadonlyRootfs}}'
# 输出: CapDrop:[ALL] ReadOnly:true
```

**4. 审计日志**：

```bash
yum install -y audit
cat > /etc/audit/rules.d/docker.rules << 'EOF'
-w /usr/bin/docker -p wa -k docker_actions
-w /var/lib/docker -p wa -k docker_changes
-w /etc/docker -p wa -k docker_config
EOF
systemctl restart auditd
```

## 三、iptables 安全策略

### 📍 执行位置：.20 和 .40（两台都要做）

```bash
# 放行已建立连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
# 内网 SSH 白名单
iptables -A INPUT -s 192.168.133.0/24 -p tcp --dport 22 -j ACCEPT
# Docker 内部通信
iptables -A INPUT -s 172.17.0.0/16 -j ACCEPT
# SSH 暴力破解防护
iptables -A INPUT -p tcp --dport 22 -m recent --set --name ssh_brute
iptables -A INPUT -p tcp --dport 22 -m recent --rcheck --seconds 60 --hitcount 3 --name ssh_brute -j DROP
# 默认拒绝
iptables -A INPUT -j LOG --log-prefix "DROP: "
iptables -A INPUT -j DROP

# 保存
mkdir -p /etc/iptables
iptables-save > /etc/iptables/rules.v4
```

## 四、Ansible 自动化

### 📍 执行位置：.10 docker-master

**1. 安装**：

```bash
yum install -y epel-release ansible
```

**2. 配置 Inventory** (`/etc/ansible/hosts`)：

```ini
[docker_nodes]
docker-node1 ansible_host=192.168.133.20
docker-node2 ansible_host=192.168.133.40

[all:vars]
ansible_user=root
```

**3. 验证**：

```bash
ansible docker_nodes -m ping
# docker-node1 | SUCCESS => {"ping": "pong"}
# docker-node2 | SUCCESS => {"ping": "pong"}
```

**4. 执行安全基线 Playbook**：

此 Playbook 自动推送到 .20 和 .40，完成：
- SSH 加固（禁 root 密码登录、空闲超时）
- 内核安全参数（SYN flood 防护、禁 ICMP 重定向）
- 密码策略（90天过期、最小8位）
- Docker 配置检查
- 端口收集

```bash
ansible-playbook security_baseline.yml
```

## 五、Zabbix + Grafana 监控

### 📍 执行位置：.10 docker-master

**1. 安装 Zabbix Server**：

```bash
# MariaDB
yum install -y mariadb-server && systemctl start mariadb
mysql -u root << 'SQL'
CREATE DATABASE zabbix CHARACTER SET utf8 COLLATE utf8_bin;
GRANT ALL ON zabbix.* TO zabbix@localhost IDENTIFIED BY 'zabbix123';
SQL

# Zabbix 5.0 LTS
rpm -Uvh https://repo.zabbix.com/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm
yum install -y zabbix-server-mysql zabbix-agent zabbix-web-mysql
zcat /usr/share/doc/zabbix-server-mysql-*/create.sql.gz | mysql -u zabbix -pzabbix123 zabbix

sed -i 's/;date.timezone =/date.timezone = Asia\/Shanghai/' /etc/php.ini
systemctl start zabbix-server zabbix-agent httpd
systemctl enable zabbix-server zabbix-agent httpd
```

Zabbix Web: `http://192.168.133.10:8081/zabbix`（Admin / zabbix）

**2. 安装 Grafana**：

```bash
yum install -y grafana
systemctl start grafana-server && systemctl enable grafana-server
```

Grafana: `http://192.168.133.10:3000`（admin / admin）

### 📍 执行位置：.20 和 .40（两台都要做）

**3. 安装 Zabbix Agent**：

```bash
rpm -Uvh https://repo.zabbix.com/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm
yum install -y zabbix-agent
sed -i 's/^Server=.*/Server=192.168.133.10/' /etc/zabbix/zabbix_agentd.conf
sed -i 's/^ServerActive=.*/ServerActive=192.168.133.10/' /etc/zabbix/zabbix_agentd.conf
systemctl restart zabbix-agent && systemctl enable zabbix-agent
```
