## 第二步：.10 —— 配置 SSH 免密

```bash
su -
# 密码: root

# 生成密钥
ssh-keygen -t rsa -b 2048 -f /root/.ssh/id_rsa -N ""

# 分发到另外两台（需要输入对方 root 密码）

# 验证

ssh root@docker-node2 "hostname"   # 输出 docker-node2
```

## 第三步：.20 和 .40 —— Docker 安全配置

### 两台都做

**1. 守护进程加固**：

```bash
su -
cat > /etc/docker/daemon.json << 'JSON'
{
  "icc": false,
  "live-restore": true,
  "userland-proxy": false,
  "no-new-privileges": true,
  "log-driver": "json-file",
  "log-opts": { "max-size": "10m", "max-file": "3" }
}
JSON
systemctl daemon-reload
systemctl restart docker
```

| 参数 | 作用 |
|------|------|
| icc=false | 禁止容器间直接通信 |
| no-new-privileges | 防 setuid 提权 |

**2. 启动安全容器**：

```bash
docker rm -f webapp 2>/dev/null
docker run -d --name webapp \
  --memory="256m" \
  --cpus="0.5" \
  --cap-drop=ALL \
  --cap-add=NET_BIND_SERVICE \
  --read-only \
  --tmpfs /tmp:rw,noexec,nosuid,size=64M \
  --tmpfs /run:rw,noexec,nosuid,size=64M \
  --tmpfs /var/cache/nginx:rw,noexec,nosuid,size=64M \
  --security-opt=no-new-privileges \
  -p 8080:80 \
  nginx:alpine
```

| 参数 | 安全效果 |
|------|----------|
| --memory=256m | cgroup 防 OOM 耗尽宿主机 |
| --cap-drop=ALL | 移除所有内核能力 |
| --read-only | 文件系统不可篡改 |
| --tmpfs noexec,nosuid | 临时目录禁执行 |

**3. 验证**：

```bash
docker ps
docker inspect webapp --format 'CapDrop:{{.HostConfig.CapDrop}} ReadOnly:{{.HostConfig.ReadonlyRootfs}}'
# 输出: CapDrop:[ALL] ReadOnly:true
```

**4. 审计日志**：

```bash
yum install -y audit
cat > /etc/audit/rules.d/docker.rules << 'EOF'
-w /usr/bin/docker -p wa -k docker_actions
-w /var/lib/docker -p wa -k docker_changes
-w /etc/docker -p wa -k docker_config
EOF
systemctl restart auditd
```

## 第四步：.20 和 .40 —— iptables 安全策略

**两台都做**：

```bash
# 放行已有连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
# 内网 SSH
iptables -A INPUT -s 192.168.133.0/24 -p tcp --dport 22 -j ACCEPT
# Docker 内部
iptables -A INPUT -s 172.17.0.0/16 -j ACCEPT
# SSH 暴力破解防护（60秒3次封禁）
iptables -A INPUT -p tcp --dport 22 -m recent --set --name ssh_brute
iptables -A INPUT -p tcp --dport 22 -m recent --rcheck --seconds 60 --hitcount 3 --name ssh_brute -j DROP
# 默认拒绝
iptables -A INPUT -j LOG --log-prefix "DROP: "
iptables -A INPUT -j DROP

# 保存
mkdir -p /etc/iptables
iptables-save > /etc/iptables/rules.v4
```

## 第五步：.10 —— Ansible 批量管控

```bash
su -

# 安装
yum install -y epel-release
yum install -y ansible

# 配置 inventory
cat > /etc/ansible/hosts << 'EOF'
[docker_nodes]
docker-node1 ansible_host=192.168.133.20
docker-node2 ansible_host=192.168.133.40

[all:vars]
ansible_user=root
EOF

# 验证
ansible docker_nodes -m ping
```

应该看到两个 `SUCCESS => {"ping": "pong"}`。

**执行安全基线 Playbook**——创建并运行：

```bash
cat > /root/security_baseline.yml << 'EOF'
- name: Docker 节点安全基线加固
  hosts: docker_nodes
  become: yes
  tasks:
    - name: "[SSH] 禁用 root 密码登录"
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: '^PermitRootLogin'
        line: 'PermitRootLogin without-password'
    - name: "[SSH] 禁用空密码"
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: '^PermitEmptyPasswords'
        line: 'PermitEmptyPasswords no'
    - name: "[SSH] 重启 sshd"
      systemd: name=sshd state=restarted
      ignore_errors: yes
    - name: "[PAM] 密码有效期 90 天"
      lineinfile:
        path: /etc/login.defs
        regexp: '^PASS_MAX_DAYS'
        line: 'PASS_MAX_DAYS 90'
    - name: "[KERNEL] SYN flood 防护"
      sysctl: name=net.ipv4.tcp_syncookies value=1 state=present reload=yes
    - name: "[KERNEL] 禁 IP 转发"
      sysctl: name=net.ipv4.ip_forward value=0 state=present reload=yes
    - name: "[KERNEL] 拒绝 ICMP 重定向"
      sysctl: name=net.ipv4.conf.all.accept_redirects value=0 state=present reload=yes
    - name: "[DOCKER] 检查运行状态"
      systemd: name=docker state=started enabled=yes
    - name: "[DOCKER] 输出容器安全配置"
      shell: for c in $(docker ps -q); do docker inspect --format '{{.Name}}: CapDrop={{.HostConfig.CapDrop}} ReadOnly={{.HostConfig.ReadonlyRootfs}} Mem={{.HostConfig.Memory}}' $c; done
      register: result
    - debug: var=result.stdout_lines
    - name: "[PORT] 收集对外端口"
      shell: ss -tlnp | grep -v '127.0.0.1\|::1'
      register: ports
    - debug: var=ports.stdout_lines
EOF

ansible-playbook /root/security_baseline.yml
```

## 第六步：.10 —— 部署 Zabbix 监控

```bash
# MariaDB
yum install -y mariadb-server
systemctl start mariadb
mysql -u root << 'SQL'
CREATE DATABASE zabbix CHARACTER SET utf8 COLLATE utf8_bin;
GRANT ALL ON zabbix.* TO zabbix@localhost IDENTIFIED BY 'zabbix123';
SQL

# Zabbix 5.0
rpm -Uvh https://repo.zabbix.com/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm
yum install -y zabbix-server-mysql zabbix-agent zabbix-web-mysql
zcat /usr/share/doc/zabbix-server-mysql-*/create.sql.gz | mysql -u zabbix -pzabbix123 zabbix
sed -i 's/;date.timezone =/date.timezone = Asia\/Shanghai/' /etc/php.ini
systemctl start zabbix-server zabbix-agent httpd
systemctl enable zabbix-server zabbix-agent httpd
```

Zabbix Web 访问：`http://192.168.133.10:8081/zabbix`（Admin / zabbix）
