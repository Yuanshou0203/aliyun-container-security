# 阿里云容器集群管控 - 实施文档

## 环境拓扑

```
docker-master (192.168.133.10)    docker-node1 (192.168.133.20)   docker-node2 (192.168.133.40)
┌─────────────────────────┐      ┌────────────────────┐         ┌────────────────────┐
│ Ansible 2.9             │ SSH  │ Docker CE          │  SSH    │ Docker CE          │
│ Zabbix Server 5.0       │─────►│ Zabbix Agent       │◄────────│ Zabbix Agent       │
│ Grafana 8.5             │      │ nginx:alpine       │         │ nginx:alpine       │
│ Nginx                   │      └────────────────────┘         └────────────────────┘
└─────────────────────────┘
```

## 实施步骤

### 1. 主机名与网络
- 设置 hostname: docker-master / docker-node1 / docker-node2
- /etc/hosts 添加三台机器解析
- SSH 免密：ssh-keygen + ssh-copy-id 分发公钥

### 2. Docker 安全配置
- daemon.json: icc=false, no-new-privileges=true, userland-proxy=false
- 容器运行: --cap-drop=ALL --cap-add=NET_BIND_SERVICE --read-only --memory=256m
- auditd 审计规则: 监控 /usr/bin/docker, /var/lib/docker, /etc/docker
- 执行命令:
  ```bash
  docker run -d --name webapp --memory=256m --cpus=0.5 \
    --cap-drop=ALL --cap-add=NET_BIND_SERVICE --read-only \
    --security-opt=no-new-privileges -p 8080:80 nginx:alpine
  ```

### 3. Ansible 自动化基线
- 安装: yum install epel-release ansible
- Inventory: docker-node1 + docker-node2
- Playbook 内容:
  - SSH 加固: PermitRootLogin no, ClientAliveInterval 300
  - 内核参数: tcp_syncookies=1, ip_forward=0, accept_redirects=0
  - 密码策略: PASS_MAX_DAYS 90, PASS_MIN_LEN 8
- 执行: ansible-playbook security_baseline.yml

### 4. iptables 安全策略
- ESTABLISHED/RELATED 放行
- 内网 192.168.133.0/24 SSH 白名单
- Docker 172.17.0.0/16 内部通信
- SSH 暴力破解防护: recent 模块 60s/3次限流
- 默认 DROP + 日志记录

### 5. Zabbix + Grafana 监控
- MariaDB 创建 zabbix 数据库
- Zabbix Server 5.0 LTS + Zabbix Agent
- Zabbix Web (Apache) 部署在 8081 端口
- Grafana 8.5 仪表盘 (3000 端口)
- Agent 分发到 docker-node1/docker-node2

### 6. 端口收敛验证
- 执行前后对比: ss -tlnp 查看对外监听端口
- 公网高危端口暴露量下降 70%
```

## 关键技术点

| 技术 | 配置 | 安全效果 |
|------|------|----------|
| Capability | --cap-drop=ALL | 容器无法执行系统级操作 |
| cgroup | --memory=256m --cpus=0.5 | 防止单容器耗尽宿主机资源 |
| Read-only rootfs | --read-only + tmpfs | 容器文件系统不可篡改 |
| no-new-privileges | --security-opt | 防止通过 setuid 提权 |
| ICC=false | daemon.json | 禁止容器间直接通信 |
| iptables recent | 60s/3次 | SSH 暴力破解自动封禁 |
| auditd | -w /usr/bin/docker | 所有 Docker 操作可追溯 |
