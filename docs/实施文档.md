
**原理**：公钥认证替代密码登录，既安全又支持 Ansible 批量管控。

## 二、Docker 容器安全管控

### 2.1 Docker 守护进程加固

`/etc/docker/daemon.json`：
```json
{
  "icc": false,
  "live-restore": true,
  "userland-proxy": false,
  "no-new-privileges": true,
  "log-driver": "json-file",
  "log-opts": { "max-size": "10m", "max-file": "3" }
}
```

| 参数 | 作用 |
|------|------|
| icc=false | 禁止容器间直接通信，防止横向移动 |
| no-new-privileges | 容器内进程无法通过 setuid 提权 |
| userland-proxy=false | 减少攻击面，使用 iptables 转发 |
| live-restore | Docker 重启时容器保持运行 |

```bash
systemctl daemon-reload && systemctl restart docker
```

### 2.2 安全容器启动

```bash
docker run -d --name webapp \
  --memory="256m" \
  --cpus="0.5" \
  --cap-drop=ALL \
  --cap-add=NET_BIND_SERVICE \
  --read-only \
  --tmpfs /tmp:rw,noexec,nosuid,size=64M \
  --tmpfs /run:rw,noexec,nosuid,size=64M \
  --tmpfs /var/cache/nginx:rw,noexec,nosuid,size=64M \
  --security-opt=no-new-privileges \
  -p 8080:80 \
  nginx:alpine
```

| 参数 | 安全效果 |
|------|----------|
| --memory=256m | cgroup 限制，防止 OOM 攻击 |
| --cpus=0.5 | 限制 CPU，防止挖矿/拒绝服务 |
| --cap-drop=ALL | 移除所有 Linux Capability |
| --cap-add=NET_BIND_SERVICE | 仅保留绑定端口能力 |
| --read-only | 容器文件系统只读 |
| --tmpfs noexec,nosuid | 临时目录禁止执行文件、禁止 suid |
| --security-opt=no-new-privileges | 防止子进程获得额外权限 |

验证：
```bash
docker inspect webapp --format 'CapDrop:{{.HostConfig.CapDrop}}'
# 输出: CapDrop:[ALL]

docker inspect webapp --format 'ReadOnly:{{.HostConfig.ReadonlyRootfs}}'
# 输出: ReadOnly:true
```

### 2.3 审计日志

```bash
yum install -y audit
cat > /etc/audit/rules.d/docker.rules << 'EOF'
-w /usr/bin/docker -p wa -k docker_actions
-w /var/lib/docker -p wa -k docker_changes
-w /etc/docker -p wa -k docker_config
-w /usr/lib/systemd/system/docker.service -p wa -k docker_service
EOF
systemctl restart auditd
```

## 三、Ansible 自动化基线加固

### 3.1 安装 Ansible

```bash
yum install -y epel-release
yum install -y ansible
```

### 3.2 Inventory 配置

`/etc/ansible/hosts`：
```ini
[docker_nodes]
docker-node1 ansible_host=192.168.133.20
docker-node2 ansible_host=192.168.133.40

[all:vars]
ansible_user=root
```

### 3.3 安全基线 Playbook

`security_baseline.yml` 包含 5 类检查：

**SSH 加固**：
- PermitRootLogin without-password（禁止 root 密码登录）
- PermitEmptyPasswords no
- ClientAliveInterval 300（5分钟无操作断连）

**内核安全参数**（sysctl）：
- net.ipv4.tcp_syncookies=1 → 防 SYN Flood
- net.ipv4.ip_forward=0 → 禁止 IP 转发
- net.ipv4.conf.all.accept_redirects=0 → 拒绝 ICMP 重定向
- net.ipv4.conf.all.send_redirects=0 → 不发送 ICMP 重定向
- kernel.dmesg_restrict=1 → 限制 dmesg 访问

**密码策略**：
- PASS_MAX_DAYS 90
- PASS_MIN_LEN 8

**Docker 检查**：
- daemon.json 配置审计
- 运行容器状态采集
- 容器安全配置验证（CapDrop、ReadOnly、Memory）

**端口检查**：
- 收集所有对外监听端口

```bash
ansible-playbook security_baseline.yml
```

验证：
```bash
ansible docker_nodes -m ping
# docker-node1 | SUCCESS => { "ping": "pong" }
# docker-node2 | SUCCESS => { "ping": "pong" }
```

## 四、iptables 安全策略

```bash
# 放行已建立连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 内网 SSH 白名单
iptables -A INPUT -s 192.168.133.0/24 -p tcp --dport 22 -j ACCEPT

# Docker 内部通信
iptables -A INPUT -s 172.17.0.0/16 -j ACCEPT

# SSH 暴力破解防护（60秒内3次则封禁）
iptables -A INPUT -p tcp --dport 22 -m recent --set --name ssh_brute
iptables -A INPUT -p tcp --dport 22 -m recent --rcheck --seconds 60 --hitcount 3 --name ssh_brute -j DROP

# 默认拒绝 + 日志
iptables -A INPUT -j LOG --log-prefix "IPTABLES-DROP: " --log-level 4
iptables -A INPUT -j DROP

# 持久化
iptables-save > /etc/iptables/rules.v4
```

## 五、Zabbix + Grafana 监控

### 5.1 Zabbix Server 安装（.10）

```bash
# MariaDB 后端
yum install -y mariadb-server
systemctl start mariadb
mysql -u root << 'SQL'
CREATE DATABASE zabbix CHARACTER SET utf8 COLLATE utf8_bin;
GRANT ALL ON zabbix.* TO zabbix@localhost IDENTIFIED BY 'zabbix123';
SQL

# Zabbix 5.0 LTS
rpm -Uvh https://repo.zabbix.com/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm
yum install -y zabbix-server-mysql zabbix-agent zabbix-web-mysql

# 导入初始数据
zcat /usr/share/doc/zabbix-server-mysql-*/create.sql.gz | mysql -u zabbix -pzabbix123 zabbix

# 配置 PHP 时区
sed -i 's/;date.timezone =/date.timezone = Asia\/Shanghai/' /etc/php.ini

# 启动
systemctl start zabbix-server zabbix-agent httpd
systemctl enable zabbix-server zabbix-agent httpd
```

访问：`http://192.168.133.10:8081/zabbix`（默认 Admin/zabbix）

### 5.2 Zabbix Agent 分发

```bash
# 在 docker-node1 和 docker-node2 上
rpm -Uvh https://repo.zabbix.com/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm
yum install -y zabbix-agent
sed -i 's/^Server=.*/Server=192.168.133.10/' /etc/zabbix/zabbix_agentd.conf
sed -i 's/^ServerActive=.*/ServerActive=192.168.133.10/' /etc/zabbix/zabbix_agentd.conf
systemctl restart zabbix-agent
systemctl enable zabbix-agent
```

### 5.3 Grafana

```bash
yum install -y grafana
systemctl start grafana-server
systemctl enable grafana-server
```

访问：`http://192.168.133.10:3000`（默认 admin/admin）
